# Nornyx Product Thesis — Short Version

Nornyx is a context-native agentic engineering language.

It exists to make AI-assisted engineering governable, auditable, repeatable, and easier for both humans and models.

## Core thesis

```text
One .nyx source should declare the AI engineering contract.
Generated downstream artifacts should serve existing repos, IDEs, agents, CI, PMO, and evidence workflows.
```

## Nornyx should reduce

```text
scattered instructions
duplicated prompts
skills sprawl
agent config drift
context chaos
manual evidence packs
manual PMO status
unbounded agent execution
unclear approvals
inconsistent IDE/repo rules
```

## Nornyx should not become

```text
another endless config pile
a premature Python replacement
a hidden autonomous execution system
a portal-first product
a proprietary trap around existing tools
```

## Product boundary

Add features only when they:

```text
reduce duplication
generate standard artifacts
improve validation
improve safety
improve evidence
improve human/model usability
```

## One-line identity

```text
Nornyx is the executable contract layer for human-model software delivery.
```
