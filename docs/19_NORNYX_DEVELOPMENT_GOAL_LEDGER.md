# Nornyx Development Goal Ledger

This ledger turns the full Nornyx roadmap into executable goals.

| Goal | Phase | Outcome |
|---|---|---|
| GOAL-000 | v0.1 | Freeze positioning: control-plane language, not GP language replacement |
| GOAL-001 | v0.1 | Freeze core syntax and examples |
| GOAL-002 | v0.1 | Harden parser/checker diagnostics |
| GOAL-003 | v0.1 | Generate compatibility artifacts: AGENTS, skills, harness, policy, eval, evidence |
| GOAL-004 | v0.2 | Add formal grammar and schema model |
| GOAL-005 | v0.2 | Add context provenance, taint, authority order, channel separation |
| GOAL-006 | v0.3 | Implement harness runtime with bounded repair loop |
| GOAL-007 | v0.3 | Implement evidence/trace runtime |
| GOAL-008 | v0.5 | Add policy, guardrail, capability, connector safety |
| GOAL-009 | v0.5 | Add eval runner and eval integrity |
| GOAL-010 | v0.7 | Add plugin/extension system and MCP/A2A adapters |
| GOAL-011 | v0.8 | Add LSP/editor support |
| GOAL-012 | v1.0 | Release stable LLM-native engineering control-plane language |
| GOAL-013 | future | Explore full programming-language constructs and native backends |

## Execution rule

Every goal must have:

- scope;
- non-goals;
- validation commands;
- evidence path;
- approval gates;
- stop rules;
- model routing;
- rollback/next-step plan.
